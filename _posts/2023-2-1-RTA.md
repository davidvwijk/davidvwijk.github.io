# Background

\section{Background}

Run time assurance (RTA), is a general framework that monitors the desired control actions of a known or unknown primary controller, and can intervene when necessary to ensure safety of the system. This is a very powerful tool for autonomous systems and can be employed if a few assumptions about the system are made, and the notion of safety is carefully defined. In this short summary, I will cover control barrier functions (CBF's) and an application of CBF's known as active set invariance filters (ASIF). The summary is primarily based on the following references: [1], [2].

In fact, a form of RTA was used in AFRL's Automatic Ground Collision Avoidance System (Auto-GCAS). It uses inputs from terrain mapping, aircraft location, and automation to avoid ground collisions, intervening when absolutely necessary in order to save the lives of pilots. \section{Preliminaries}

\noindent \textbf{Control affine dynamics}:

$$ \begin{align} \label{eq: dynamics}
    \dot{\boldsymbol{x}}(t) = \boldsymbol{f}(x) + \boldsymbol{g}(x)\boldsymbol{u}
\end{align} $$

where both $$\boldsymbol{f}(x)$$ and $$\boldsymbol{g}(x)$$ must be Lipschitz continuous functions, meaning that each function that is defined on an interval and has a bounded first derivative. \\
\noindent \textbf{Forward Invariance}: A set $$\mathcal{C}_S$$ is \textit{forward invariant} for a dynamical system if every solution to the system from the set $$\mathcal{C}_S$$ stays in set. Mathematically, 

$$\begin{equation}
    \boldsymbol{x}(t_0) \in \mathcal{C}_S \Rightarrow \boldsymbol{x}(t) \in \mathcal{C}_S, \forall t \ge t_0
\end{equation}$$

\noindent \textbf{Control Invariance}: A set $$\mathcal{S} \subseteq X$$ is \textit{control invariant} for a dynamical system if there exists an admissible control input $$\boldsymbol{u} \in \mathcal{U}$$ for every $$\boldsymbol{x} \in \mathcal{S}$$ such $$\boldsymbol{x}$$ remains in $$\mathcal{S}$$ for all positive time (i.e. renders $$\mathcal{S}$$ forward invariant). 

\subsection{Safety}

In the following formulations, safety is defined as a subset of the possible states the dynamical system can be described by. As such, safety is enforced using safety constraints, which are inequality constraints on functions of the state. The constraints take the form 

$$ \begin{align}
    \varphi_i(\boldsymbol{x}): \mathbb{R}^n \Rightarrow \mathbb{R}, ~ \varphi_i(\boldsymbol{x}) \ge 0, ~ \forall i \in \{1,...,N\}
\end{align} $$

\noindent for $$N$$ constraints. The subset of all possible states that satisfies all of the safety constraints, $$\varphi(x)$$, is referred to as the constraint set, or the allowable set,

$$ \begin{equation}
    \mathcal{C}_A := \{\boldsymbol{x} \in \mathbb{R} ~|~ \varphi_i(\boldsymbol{x}) \ge 0, ~ \forall i \in \{1,...,N\} \}
\end{equation} $$
